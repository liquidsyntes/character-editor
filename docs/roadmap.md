## Как читать этот роадмап

Каждая задача — готовая карточка для Antigravity. Формат:
- **ID** — для ссылок в зависимостях
- **Источник** — откуда взята задача (audit / feature-ideas)
- **Приоритет** — 🔴 блокирует другие задачи / 🟡 важно для роста / 🟢 nice-to-have
- **Оценка** — в часах
- **Зависимости** — какие задачи должны быть сделаны до этой
- **Файлы** — что трогать

## Фаза 2: Стабилизация AI-подсистемы

Самый чувствительный к багам слой.

### 2.1 🟡 Протестировать DiffModal на разных AI-провайдерах
- **Источник:** audit #5
- **Оценка:** 2 ч
- **Зависимости:** нужны реальные API-ключи для xAI, Anthropic, OpenAI
- **Файлы:** `src/components/DiffModal.tsx`, ручное тестирование
- **Суть:** Разные модели возвращают разные диалекты Markdown — `diffWords` из пакета `diff` может ломаться на нестандартном форматировании. Прогнать fill → diff на DeepSeek, Grok, Claude, GPT. Чек-лист: жирный текст, списки, кавычки, спецсимволы в ответах.
- **Критерий готовности:** DiffModal корректно показывает изменения от всех 4 провайдеров; баги задокументированы в `docs/diff-provider-compatibility.md`.

### 2.2 🟡 Унифицировать SSE-логику (убрать дублирование)
- **Источник:** feature-ideas 2 (техдолг)
- **Оценка:** 1.5 ч
- **Зависимости:** нет
- **Файлы:** `src/hooks/useAiFill.ts`, `src/hooks/useCharacterAnalysis.ts` → использовать `src/lib/ai/streamUtils.ts`
- **Суть:** SSE-стриминг написан 4 раза: в `fill/route.ts`, `analyze/route.ts` (сервер) и в `useAiFill.ts`, `useCharacterAnalysis.ts` (клиент). Серверная часть уже вынесена в `streamUtils.ts`. Клиентскую — использовать `fetchSseStream` из `prompt-parser.ts`.
- **Критерий готовности:** в `useAiFill.ts` и `useCharacterAnalysis.ts` нет инлайн-цикла `while (true) { reader.read() }`.

### 2.3 🟡 Единый source of truth для AI-моделей
- **Источник:** feature-ideas 2 (техдолг)
- **Оценка:** 1 ч
- **Зависимости:** нет
- **Файлы:** `src/lib/ai/provider.ts`, `src/lib/ai/useAiSettings.ts` → новый API-роут `GET /api/ai/models`
- **Суть:** Список моделей прописан в ДВУХ файлах: `provider.ts` (сервер) и `useAiSettings.ts` (клиент). Сделать `GET /api/ai/models` → клиент получает список оттуда.
- **Критерий готовности:** добавил модель в `provider.ts` → она появилась в UI без правки `useAiSettings.ts`.

## Фаза 5: Новые фичи — связи и контекст

### 5.1 🟡 Связи между персонажами
- **Источник:** feature-ideas 3.1
- **Оценка:** 6 ч
- **Зависимости:** 1.2 (JSON-схема), 1.4 (рефакторинг CharacterForm)
- **Файлы:** `prisma/schema.prisma` (новая модель `CharacterRelation`), новый компонент `RelationGraph.tsx`, `CharacterForm.tsx` (блок «Связи»)
- **Суть:** Персонажи в проекте связываются: родство, дружба, вражда, романтика, подчинение. Визуальный граф на SVG с draggable nodes. AI-генерация отношений.
- **Критерий готовности:** можно создать связь между двумя персонажами; граф отображается в проекте и в карточке.

### 5.2 🟡 Таймлайн жизни персонажа
- **Источник:** feature-ideas 3.2
- **Оценка:** 5 ч
- **Зависимости:** 1.2, 1.4
- **Файлы:** `prisma/schema.prisma` (новая модель `TimelineEvent`), `TimelineView.tsx`, API-роут `/api/ai/timeline`
- **Суть:** Хронологическая лента событий. AI генерирует ключевые события на основе заполненных полей.
- **Критерий готовности:** у персонажа есть таймлайн с 5+ событиями; AI предлагает события по кнопке.

### 5.3 🟡 Сравнение персонажей + архетипы
- **Источник:** feature-ideas 3.4
- **Оценка:** 3 ч
- **Зависимости:** 1.4
- **Файлы:** новый компонент `CompareView.tsx`, API-роут `/api/ai/archetype`
- **Суть:** Side-by-side сравнение двух персонажей с подсветкой различий. AI классифицирует по архетипам (Герой, Трикстер, Ментор, etc.).
- **Критерий готовности:** выбор двух персонажей → сравнение в таблице; AI возвращает архетип и обоснование.
