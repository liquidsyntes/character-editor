## Методология работы

- **Одна задача = одна ветка** (`feat/...`, `fix/...`, `chore/...`)
- **Не коммитить в main**
- **Перед PR:** `npm run build && npm run lint && npm test`
- **После вливания:** удалить ветку, вычеркнуть задачу из этого плана
- **План обновлять** раз в неделю или после крупных изменений
