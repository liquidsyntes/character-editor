(разбить гигантский файл CharacterForm.tsx размером ~25 КБ на изолированные логические субкомпоненты по секциям персонажа — AppearanceSection, BackgroundSection и т.д., чтобы упростить поддержку и управление состоянием React-компонента)

(создать строгую JSON Schema или отдельный документ, описывающий структуру поля data со всеми 145 характеристиками, чтобы застраховаться от поломки старых карточек при добавлении новых полей в будущем)

(разобраться с жесткой привязкой к бинарнику @libsql/linux-x64-gnu, настроить package.json и Docker-сборку так, чтобы Prisma стабильно работала без ручных установок пакетов при смене операционной системы или деплое)

(провести ручное тестирование компонента DiffModal на ответах от разных AI-провайдеров — DeepSeek, xAI, Anthropic — так как модели используют разные диалекты Markdown, что может ломать визуальное отображение word-level диффов перед сохранением)

6. Настройка CI/CD пайплайна:
(настроить GitHub Actions или GitLab CI для автоматического запуска имеющихся тестов Vitest, линтера ESLint и сборки npm run build при каждом пуше или пулл-реквесте)

7. Аудит инструкций для AI-агентов:
(актуализировать файлы AGENTS.md, GEMINI.md, CLAUDE.md, добавив туда информацию о новых архитектурных правилах, чтобы будущие AI-ассистенты, работающие с кодом, не ломали внедренные улучшения)
